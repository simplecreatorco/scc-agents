Part 1
One-Time Setup Do these steps once — you'll never need to repeat them
Step 1 Install Claude Code 0 / 6 ▾

Claude Code is the desktop app you'll use to run everything. It's included in your Max plan — no extra cost.

Download & Install
Open Safari or Chrome on your Mac
Go to claude.ai/download and download the Mac version (file ends in .dmg)
Open your Downloads folder, double-click the .dmg file, then drag Claude into your Applications folder Finder
Open Claude from Applications. If Mac asks "Are you sure?" — click Open
Log in with the same Anthropic account as your Max plan subscription (don't create a new account)
Test: type "Hello" in the chat and press Enter — Claude should reply ✓ Verify
✅ No extra billing setup needed

Claude Code is included in your Max plan. No Anthropic Console account, no API credits, no separate charges.

Step 2 Create the Project Folder Structure 0 / 5 ▾

All pipeline files live in one folder on your Mac. Use Terminal to create everything at once — it's faster and avoids hidden-folder issues.

⚠️ Folders starting with a dot (.claude) are hidden on Mac

After creating the folders, press Cmd + Shift + . in Finder to show hidden files. Press the same keys again to hide them. Claude Code will still see them regardless.

Create all folders with one Terminal command
Open Terminal: press Cmd + Space, type Terminal, press Enter Terminal
Paste and run this command (creates all folders at once): 
mkdir -p ~/Documents/scc-blog-pipeline/.claude/agents \
  ~/Documents/scc-blog-pipeline/.claude/commands \
  ~/Documents/scc-blog-pipeline/data \
  ~/Documents/scc-blog-pipeline/logs
Verify it worked — run: ls ~/Documents/scc-blog-pipeline/ — you should see .claude data logs Verify
In Finder, navigate to Documents → scc-blog-pipeline. Press Cmd+Shift+. to confirm .claude folder is visible Finder
Confirm final structure: scc-blog-pipeline/ contains .claude/agents/, .claude/commands/, data/, and logs/ Verify

Step 4 Create the wp-term-cache.json File 0 / 2 ▾

This file starts empty and the pipeline fills it automatically on first run. It caches your WordPress category and tag IDs so it doesn't re-query them every time.

In Terminal, run this command to create the empty file: 
echo '{}' > ~/Documents/scc-blog-pipeline/data/wp-term-cache.json
Verify it worked: run cat ~/Documents/scc-blog-pipeline/data/wp-term-cache.json — you should see {} Verify
Step 5 Create the .env File 0 / 3 ▾

The .env file stores your WordPress credentials securely. The pipeline reads it automatically each run. Think of it as a keychain file — you set it once and forget about it.

In Terminal, create the .env file (replace placeholder values with your real credentials): 
cat > ~/Documents/scc-blog-pipeline/.env << 'EOF'
WP_SITE_URL=https://thesimplecreator.com
WP_USERNAME=your_wordpress_username_here
WP_APP_PASSWORD=your_app_password_here
EOF
Verify: run cat ~/Documents/scc-blog-pipeline/.env — you should see all 3 lines Verify
After getting your App Password (Step 6), update the file: nano ~/Documents/scc-blog-pipeline/.env — edit, then Ctrl+X → Y → Enter Terminal
⚠️ Never share this file

The .env file contains your WordPress password. Never email it, upload to Google Drive, or share with anyone. If you ever need to move the pipeline to a new computer, recreate the file from scratch.

Step 6 Get Your WordPress Application Password 0 / 6 ▾

An Application Password is a special one-purpose password just for apps — completely separate from your regular WordPress login. Even if someone got it, they couldn't log into your dashboard with it.

Create the Application Password
Go to thesimplecreator.com/wp-admin and log in normally WordPress
In the left sidebar: hover over Users → click Profile WordPress
Scroll all the way down to the Application Passwords section WordPress
In the name field, type: SCC Blog Pipeline — then click Add New Application Password WordPress
Copy the generated password immediately — it looks like: xxxx xxxx xxxx xxxx xxxx xxxx ⚠️ Shows Once
Paste the password into your .env file: open Terminal → nano ~/Documents/scc-blog-pipeline/.env → replace your_app_password_here → Ctrl+X → Y → Enter Terminal
⚠️ WordPress only shows this password once

If you miss it, you'll need to delete it in WordPress and create a new one. Copy it before clicking anything else.

Step 7 Confirm Yoast SEO is Active 0 / 2 ▾

The pipeline writes Yoast SEO metadata for every post. If Yoast isn't active, the metadata fields won't populate and you'll lose the SEO advantage.

In WordPress admin: click Plugins → Installed Plugins → find Yoast SEO in the list WordPress
Confirm it shows as Active. If not: click Activate. If it's missing entirely: click Add New, search Yoast SEO, install and activate. Verify
Step 8 Verify Your Author Page Exists 0 / 1 ▾

Every post includes author schema markup that links to your About page. This helps AI tools recognise you as the named expert and authority. If the page isn't live, the schema breaks.

Open a browser and visit thesimplecreator.com/about — the page should load publicly with no errors. If it's not published, go to WordPress and publish it. Verify
Step 9 Disable Mac Sleep 0 / 3 ▾

If you're running the pipeline on a schedule overnight, your Mac must stay awake. If it sleeps, the scheduled run won't happen.

Apple menu (top-left) → System Settings → Displays → set "Turn off display after" to Never or a long duration
Go back → Battery → Options → enable "Prevent Mac from sleeping automatically when the display is off"
Plug your Mac in before leaving it overnight — battery mode may override these settings ⚠️ Important

Confirm the file browser on the left shows .claude, data, and logs folders Verify
Quick test: type /run-aeo-pipeline in the chat. Claude Code should recognise it as a command. (Don't let it run yet — just confirm it's recognised.) Verify
✅ Setup is complete!

Step 12 Check Usage After the First Run 0 / 2 ▾

The pipeline runs on your Max plan's usage allocation. Check how much one post consumed — this tells you how many posts you can comfortably run per batch or per week.

In Claude Code, type: /status and press Enter to see usage consumed
Note how much one post used. If usage is low: you can run 3–5 posts per batch. If higher: run posts on separate days.

Step 14 Review and Publish Your First Draft 0 / 5 ▾

All posts are created as drafts — you always review before publishing. The whole review process should take 5–10 minutes per post.

Go to: thesimplecreator.com/wp-admin/edit.php?post_status=draft&post_type=post WordPress
Click the new draft to open it in the editor
Read the SCC AEO Pipeline — Editor Note at the very top of the post (30 seconds — tells you the target question, competition, and why this post wins)
Skim the post — it should sound like you. Make any edits you want.
Click Publish when you're happy WordPress
✅ The editor note won't appear on the live post

It's only visible inside your WordPress drafts editor. Readers never see it.

Part 3
Scheduling Overnight Runs Set it once and wake up to new drafts
Step 15 Create the Schedule File 0 / 4 ▾

Your Mac has a built-in task scheduler called launchd — like setting an alarm that runs a program instead of playing a sound. You create one small settings file and it handles everything.

⚠️ Replace /Users/rebeccarice with your actual home folder path

First run echo $HOME in Terminal to get your exact path. Replace every instance of /Users/rebeccarice in the command below with what you see.

Run echo $HOME in Terminal and note your home path Terminal
Create the schedule file — paste this command in Terminal (with your home path substituted): 
cat > ~/Library/LaunchAgents/com.scc.blog-pipeline.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0"><dict>
  <key>Label</key>
  <string>com.scc.blog-pipeline</string>
  <key>ProgramArguments</key>
  <array>
    <string>/usr/local/bin/claude</string>
    <string>--project</string>
    <string>/Users/rebeccarice/Documents/scc-blog-pipeline</string>
    <string>/run-aeo-pipeline</string>
    <string>1</string>
  </array>
  <key>WorkingDirectory</key>
  <string>/Users/rebeccarice/Documents/scc-blog-pipeline</string>
  <key>StartCalendarInterval</key>
  <array>
    <dict><key>Hour</key><integer>23</integer>
          <key>Minute</key><integer>0</integer>
          <key>Weekday</key><integer>1</integer></dict>
    <dict><key>Hour</key><integer>23</integer>
          <key>Minute</key><integer>0</integer>
          <key>Weekday</key><integer>2</integer></dict>
    <dict><key>Hour</key><integer>23</integer>
          <key>Minute</key><integer>0</integer>
          <key>Weekday</key><integer>3</integer></dict>
    <dict><key>Hour</key><integer>23</integer>
          <key>Minute</key><integer>0</integer>
          <key>Weekday</key><integer>4</integer></dict>
    <dict><key>Hour</key><integer>23</integer>
          <key>Minute</key><integer>0</integer>
          <key>Weekday</key><integer>5</integer></dict>
  </array>
  <key>StandardOutPath</key>
  <string>/Users/rebeccarice/Documents/scc-blog-pipeline/logs/pipeline.log</string>
  <key>StandardErrorPath</key>
  <string>/Users/rebeccarice/Documents/scc-blog-pipeline/logs/pipeline-error.log</string>
</dict></plist>
EOF
The number 1 in the <string>1</string> line = posts per night. Change to 3 if you want 3 posts, etc.
This schedule runs Monday–Friday at 11 PM. Adjust Weekday numbers (1=Mon, 2=Tue ... 7=Sun) or Hour integer if you want different timing
Step 16 Activate the Schedule 0 / 3 ▾
In Terminal, run: 
launchctl load ~/Library/LaunchAgents/com.scc.blog-pipeline.plist
Verify it's active — run: launchctl list | grep scc — you should see com.scc.blog-pipeline in the output Verify
To pause the schedule any time: launchctl unload ~/Library/LaunchAgents/com.scc.blog-pipeline.plist
Step 17 Schedule the Monthly Freshness Audit 0 / 2 ▾

The freshness audit checks all your existing posts and flags anything that's gone stale — set it to run once a month automatically.

Create a second schedule file for the monthly audit (same process as Step 15 — substitute your home path). Set it to run on the first Sunday of each month at 10 PM (Hour: 22, Weekday: 1 with Day: 1–7 logic). You can also just run /run-freshness-audit manually each month instead.
Add a recurring reminder to your calendar: First Sunday of every month — Run freshness audit

Once a month, run the freshness audit to flag any posts that have gone stale, are missing AEO elements, or have outdated information.

Quick Reference

The 3 Commands
/run-aeo-pipeline Writes 1 post automatically. Add a number for more: /run-aeo-pipeline 3
/request-post "topic" Writes one specific post you've chosen. Use /request-post brainstorm "theme" to generate ideas first.
/run-freshness-audit Monthly audit of all published posts. Flags stale content and gives you specific fix instructions.
/status Check how much of your Max plan's usage the last run consumed.
Common Fixes
Won't connect to WP → Check your .env file. Confirm WP_APP_PASSWORD is the Application Password, not your login password.
Post stuck in audit loop → Check data/review-queue/ folder for the saved draft.
Usage limit reached → Wait for the 5-hour window to reset, or run /status to check remaining capacity.
Key Locations
WordPress drafts → thesimplecreator.com/wp-admin/edit.php?post_status=draft&post_type=post
Local post backups → scc-blog-pipeline/data/drafts/
Failed posts → scc-blog-pipeline/data/failed-posts/
Review queue → scc-blog-pipeline/data/review-queue/
Pipeline logs → scc-blog-pipeline/logs/pipeline.log
